# SKU 批量编辑与统一分页设计

## 目标

- SKU 映射搜索后，可在当前页勾选 `cleaned_sku` 完全相同的记录，批量修改产品类型和上品人。
- 新建 SKU 映射时，可由原始 SKU 一键生成清洗后 SKU。
- SKU 映射、产品类型和订单处理配置使用同一套分页组件，支持每页数量和指定页跳转。
- 直接整理并反馈“有 SKU 无中文名称”和“有中文名称无 SKU”的数据，不生成 Excel。

## 批量编辑

SKU 映射表增加复选框和批量修改按钮。选择仅在当前页有效，翻页、切换标签或重新查询后清空。前端只允许勾选 `cleaned_sku` 与首条记录相同的行，并在选择不一致时立即提示。

点击批量修改后打开弹窗。弹窗显示选中数量和共同的清洗后 SKU，并提供产品类型、上品人两个可搜索下拉框。提交到独立批量更新接口；服务器重新加载全部 ID，逐条校验修改权限、记录未删除、ID 数量完整且 `cleaned_sku` 唯一。通过后在事务内同步更新 `product_type_id`、中文名称快照、上品人员工 ID 和姓名快照。

## 一键清洗

仅在新增 SKU 映射表单的原始 SKU 后显示“清洗 SKU”按钮。点击后通过受登录和创建权限保护的 JSON 接口调用 `SkuCleaningService::cleanSkuUsingValuesAndPatterns()`，将结果填入清洗后 SKU 输入框，但不自动提交表单。原始 SKU 为空或清洗规则文件异常时，在当前表单显示错误。

## 分页

建立一个 Blade 分页组件，统一显示总记录数、每页 `20/50/100` 条、上一页、当前页/总页数、下一页和页码跳转。组件保留当前搜索、筛选及标签参数，并将目标页限制在有效范围内。

同一组件接收不同参数名，避免同页数据互相影响：SKU 映射使用 `sku_page`/`sku_per_page`，产品类型使用 `type_page`/`type_per_page`，订单处理配置使用 `page`/`per_page`。控制器只接受白名单中的每页数量，非法值回退到 `50`。

## 缺失数据口径

- 有 SKU 无中文名称：未软删除 SKU 映射中，产品类型为空、关联产品类型不存在/已删除，或中文名称为空的记录。
- 有中文名称无 SKU：未软删除产品类型中，没有任何未软删除 SKU 映射的记录。

实施完成后重新查询当前数据库并直接反馈结果。设计阶段快照为：前者 0 条，后者 72 条。

## 验证

- 功能测试覆盖批量更新成功、混合清洗后 SKU 拒绝、越权/缺失 ID 拒绝及事务一致性。
- 单元或功能测试覆盖一键清洗成功、空值和规则异常。
- 页面测试覆盖三个列表的每页数量、指定页跳转、筛选参数保留和分页参数互不干扰。
- 使用隔离的 SQLite 测试数据库，不对本地业务数据库执行刷新或清表。
