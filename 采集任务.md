# 采集任务命令

本文档记录 SKU 选项图片采集、采集结果修复、冲突检测和重采集相关命令。

所有命令默认在项目根目录执行：

```powershell
cd D:\workspace\shopify-workbench
```

## 命令列表

查看当前项目内所有 SKU 采集相关命令：

```powershell
php artisan list sku-options
```

查看单个命令帮助：

```powershell
php artisan sku-options:scrape --help
php artisan sku-options:conflicts --help
php artisan sku-options:repair-json --help
```

## SKU 图片采集

输入文件要求：文本文件，每行一个商品 URL。

一次性采集全部 URL：

```powershell
php artisan sku-options:scrape storage/app/private/product-urls.txt --timeout=120
```

按批次采集，每批 10 个链接：

```powershell
php artisan sku-options:scrape storage/app/private/product-urls.txt --timeout=120 --batch-size=10 --progress=storage/app/private/sku-options-progress.json
```

从指定序号继续批次采集，并追加到现有结果：

```powershell
php artisan sku-options:scrape storage/app/private/product-urls.txt --timeout=120 --batch-size=10 --start=11 --append --progress=storage/app/private/sku-options-progress.json
```

采集输出默认位置：

```text
storage/app/private/sku-options-image.json
storage/app/private/sku-options-image/
storage/app/private/sku-options-progress.json
```

## 直接运行 Node 采集器

通常优先使用 artisan 命令。只有调试底层 Playwright 采集器时才直接运行 Node 脚本。

输入 JSON 格式：

```json
{
  "urls": [
    "https://broderft.com/products/example"
  ]
}
```

命令：

```powershell
node scripts/sku-options-scraper.js --input storage/app/private/sku-options-url-batch.json --output storage/app/private/sku-options-worker-output.json --timeout 120
```

## 修复采集结果

只重写每个 product_id 内的 option sort_order：

```powershell
php artisan sku-options:repair-json --normalize-sort-order
```

按 product_id 重新采集并替换现有 JSON 中对应记录：

```powershell
php artisan sku-options:repair-json --id=528 --timeout=120
```

一次修复多个 product_id：

```powershell
php artisan sku-options:repair-json --id=528 --id=789 --timeout=120 --batch-size=10
```

重新采集之前出现 `ERR_CONNECTION_CLOSED` 的商品：

```powershell
php artisan sku-options:repair-json --retry-connection-closed --timeout=120 --batch-size=10
```

重新采集 unknown plugin 但有 SKU 的商品：

```powershell
php artisan sku-options:repair-json --retry-unknown-sku --timeout=120 --batch-size=10
```

重新采集 plugin 为 unknown 且 status 为 failed 的商品：

```powershell
php artisan sku-options:repair-json --retry-unknown-failed --timeout=120 --batch-size=10
```

组合修复：

```powershell
php artisan sku-options:repair-json --retry-connection-closed --retry-unknown-sku --retry-unknown-failed --timeout=120 --batch-size=10
```

## 冲突检测

检测所有 cleaned_sku 下的选项图片组冲突，并输出报告：

```powershell
php artisan sku-options:conflicts --output=storage/app/private/sku-options-conflict-report-current.md
```

检测单个 cleaned_sku：

```powershell
php artisan sku-options:conflicts --cleaned-sku=CS-QK0653-CX --output=storage/app/private/sku-options-conflict-report-qk0653-current.md
```

检测多个 cleaned_sku：

```powershell
php artisan sku-options:conflicts --cleaned-sku=CS-QK0007-CX --cleaned-sku=CS-QK0104-MJX --cleaned-sku=CS-QK0241 --cleaned-sku=CS-QK0653-CX --cleaned-sku=CS-QK1731 --cleaned-sku=CS-QK2579-CX --cleaned-sku=CS-QK3385-TH --cleaned-sku=CS-TH --cleaned-sku=CX-QK2387-CS --cleaned-sku=LP-CHJ8057 --output=storage/app/private/sku-options-conflict-report-current.md
```

## 冲突重采集

检测单个 cleaned_sku，并重新采集该冲突组内商品 URL 做对比：

```powershell
php artisan sku-options:conflicts --cleaned-sku=CS-QK0653-CX --rerun --timeout=120 --output=storage/app/private/sku-options-conflict-report-qk0653-rerun.md
```

检测所有冲突，并重新采集冲突商品 URL 做对比：

```powershell
php artisan sku-options:conflicts --rerun --timeout=120 --output=storage/app/private/sku-options-conflict-report-rerun.md
```

重跑指定 10 个 cleaned_sku 冲突组：

```powershell
php artisan sku-options:conflicts --cleaned-sku=CS-QK0007-CX --cleaned-sku=CS-QK0104-MJX --cleaned-sku=CS-QK0241 --cleaned-sku=CS-QK0653-CX --cleaned-sku=CS-QK1731 --cleaned-sku=CS-QK2579-CX --cleaned-sku=CS-QK3385-TH --cleaned-sku=CS-TH --cleaned-sku=CX-QK2387-CS --cleaned-sku=LP-CHJ8057 --rerun --timeout=120 --output=storage/app/private/sku-options-conflict-report-rerun.md
```

## 后台运行长任务

后台运行 10 个 cleaned_sku 冲突组重采集，并记录日志：

```powershell
$log = Join-Path (Get-Location) 'storage\app\private\sku-options-conflict-rerun.log'
$err = Join-Path (Get-Location) 'storage\app\private\sku-options-conflict-rerun.err.log'
$args = @(
    'artisan',
    'sku-options:conflicts',
    '--cleaned-sku=CS-QK0007-CX',
    '--cleaned-sku=CS-QK0104-MJX',
    '--cleaned-sku=CS-QK0241',
    '--cleaned-sku=CS-QK0653-CX',
    '--cleaned-sku=CS-QK1731',
    '--cleaned-sku=CS-QK2579-CX',
    '--cleaned-sku=CS-QK3385-TH',
    '--cleaned-sku=CS-TH',
    '--cleaned-sku=CX-QK2387-CS',
    '--cleaned-sku=LP-CHJ8057',
    '--rerun',
    '--timeout=120',
    '--output=storage/app/private/sku-options-conflict-report-rerun.md'
)
$process = Start-Process -FilePath 'php' -ArgumentList $args -WorkingDirectory (Get-Location) -RedirectStandardOutput $log -RedirectStandardError $err -WindowStyle Hidden -PassThru
"PID=$($process.Id)`nLOG=$log`nERR=$err"
```

查看后台进程：

```powershell
Get-Process php,node,chrome -ErrorAction SilentlyContinue | Select-Object Id,ProcessName,StartTime,CPU
```

查看日志文件大小和更新时间：

```powershell
Get-Item storage\app\private\sku-options-conflict-rerun.log, storage\app\private\sku-options-conflict-rerun.err.log, storage\app\private\sku-options-conflict-report-rerun.md -ErrorAction SilentlyContinue | Select-Object FullName,Length,LastWriteTime
```

实时查看日志：

```powershell
Get-Content storage\app\private\sku-options-conflict-rerun.log -Tail 50 -Wait
```

查看错误日志：

```powershell
Get-Content storage\app\private\sku-options-conflict-rerun.err.log -Tail 50
```

## 网络权限

SKU 图片采集和冲突重采集会访问商品页面和图片资源。当前建议只对以下命令前缀做持久网络授权：

```text
php artisan sku-options:scrape
php artisan sku-options:conflicts
```

`sku-options:repair-json` 在使用 `--id`、`--retry-connection-closed`、`--retry-unknown-sku`、`--retry-unknown-failed` 时也会重新访问商品页面；如果经常运行修复重采集，也可以单独为以下前缀做持久授权：

```text
php artisan sku-options:repair-json
```
